# ⚙️ 설정 파일 가이드 (config.yaml)

런처를 처음 실행하면 `config.yaml` 파일이 자동으로 생성됩니다. 이 파일은 다운로더의 "두뇌" 역할을 하며, 스케줄러의 작동 주기, 다운로드 속도, 파일 저장 위치 등을 결정합니다.

이 파일은 **메모장(Notepad)**이나 기타 텍스트 편집기로 열어서 수정할 수 있습니다.

> **⚠️ `.yaml` 파일 수정 시 절대 규칙:**
> 형식을 정확히 유지해야 합니다! 콜론(`:`) 뒤에는 반드시 띄어쓰기를 한 칸 넣고 값을 입력하세요.
> *✅ 올바른 예: `MaxConcurrentDownloads: 15`*
> *❌ 잘못된 예: `MaxConcurrentDownloads:15`*

---

## ⚙️ 1. 핵심 설정 (CORE SETTINGS)
*스위트가 게시물을 식별하고 처리하는 가장 기본적인 규칙입니다.*

* **`BoardUrl`**
  * **기본값:** `"https://gall.dcinside.com/board/lists/?id=comic_new6&exception_mode=recommend"`
  * **기능:** 스캐너가 크롤링할 대상 갤러리 주소입니다. 기본값인 `exception_mode=recommend`는 '개념글(추천글)'만 스캔한다는 뜻입니다.
* **`RequireTranslationPrefix`**
  * **기본값:** `"True"`
  * **기능:** `True`로 설정하면 제목에 "번역"이라는 단어가 들어간 글만 수집합니다. 모든 글을 긁고 싶다면 `False`로 변경하세요.
* **`ForceRedownload`**
  * **기본값:** `"False"`
  * **기능:** `True`로 설정하면 다운로드 기록을 무시합니다. 단, "스마트 업데이트" 기능이 있어 폴더가 이미 있다면 누락된(새로운) 이미지만 다운로드하며, 폴더를 직접 삭제했다면 전체를 다시 다운로드합니다.

## 🔎 2. 크롤러 및 탐색 범위 (CRAWLER & SEARCH LIMITS)
*스위트가 게시판을 얼마나 깊게 탐색할지 결정합니다.*

* **`AutoCrawlerMaxPages`**
  * **기본값:** `1`
  * **기능:** 백그라운드 스케줄러가 한 번 깨어날 때마다 최신 글을 몇 페이지까지 확인할지 결정합니다. 자주 실행되므로 1~3의 낮은 값으로 유지하는 것이 좋습니다.
* **`SeriesBrowserMaxPages`**
  * **기본값:** `10`
  * **기능:** 수동 메뉴인 "게시판 크롤러(Series Scanner)"를 실행할 때 탐색할 페이지 수입니다. 기본적으로 10페이지를 깊게 탐색하여 새로운 시리즈를 발굴합니다.
* **`KeywordSearchMaxBlocks`**
  * **기본값:** `300`
  * **기능:** "키워드 정밀 검색"의 깊이입니다. 300은 전체 과거 기록을 샅샅이 뒤지는 수치입니다. 더 빠른 검색을 원하면 `10` 정도로 낮추세요.
* **`CrawlOrder`**
  * **기본값:** `0`
  * **기능:** 크롤링 방향입니다. `0`은 과거 글부터(3페이지->1페이지) 순차적으로 수집하고, `1`은 최신 글부터(1페이지->3페이지) 수집합니다.
* **`KeepUnfinishedLinks`**
  * **기본값:** `False`
  * **기능:** `True`일 경우, 아직 다운로드하지 않은 이전 대기열을 지우지 않고 새 링크를 목록 아래에 계속 누적시킵니다.
* **`JunkSeriesTitles`**
  * **기능:** 스캐너가 만화 제목을 추출할 때 완전히 무시해 버릴 "쓰레기 단어"들의 목록입니다 (`|` 기호로 구분). (예: `단편`, `이전화`, `목차` 등)
* **`DaisyChainSeries`**
  * **기본값:** `True`
  * **기능:** "마법의 연속 탐색" 기능입니다. 본문 안에 이전 화 링크가 있다면, 스캐너가 알아서 과거 글로 건너뛰며 누락된 시리즈를 싹 긁어옵니다.

## 📥 3. 다운로더 엔진 (DOWNLOADER ENGINE)
*다운로드 속도와 파일 처리 방식을 제어합니다.*

* **`DownloadDir`**
  * **기본값:** `".\Downloads"`
  * **기능:** 다운로드한 만화 이미지가 저장되는 메인 폴더입니다. 기본적으로 프로그램이 있는 폴더 안의 `Downloads` 폴더에 저장됩니다. 외장 하드 등 절대 경로(예: `"D:\MyManga"`)로 변경할 수 있습니다.
* **`MaxConcurrentDownloads`**
  * **기본값:** `15`
  * **기능:** *정확히 동시에* 몇 장의 이미지를 다운로드할지 결정합니다. 인터넷이 느리다면 수치를 낮추세요.
* **`RateLimitSeconds`**
  * **기본값:** `2.5`
  * **기능:** 한 게시물의 처리가 끝나고 다음 글로 넘어가기 전 대기하는 시간(초)입니다. 서버 차단(IP 밴)을 방지합니다.
* **`RenameFilesSequential`**
  * **기본값:** `True`
  * **기능:** 복잡한 원본 이미지 파일명들을 보기 좋게 숫자 순서대로(`001.jpg`, `002.jpg`) 바꿉니다.
* **`CustomStripChars`**
  * **기본값:** `""`
  * **기능:** 폴더 이름에 자꾸 들어가는 거슬리는 특수 기호가 있다면 여기에 입력하여 자동 삭제되게 할 수 있습니다.
* **`ShowProgressBar`**
  * **기본값:** `False`
  * **기능:** `True`로 변경하면 콘솔 창에 진행률 표시줄을 띄웁니다. `False`로 두면 화면이 더 깔끔하고 속도가 약간 더 빠릅니다.

## 🌐 4. 네트워크 연결 (NETWORK & CONNECTION)

* **`DNSAutoRepair`**
  * **기본값:** `True`
  * **기능:** 서버가 일시적으로 연결을 거부할 때, 스크립트가 스스로 윈도우의 DNS 캐시를 초기화(Flush)하고 10초 대기하여 인터넷 오류를 자동 복구합니다.
* **`UseProxy`**
  * **기본값:** `False`
  * **기능:** `False`로 두면 시스템 프록시를 우회하여 최상의 직접 연결(Direct Connection) 다운로드 속도를 확보합니다.

## 🗂️ 5. 데이터 경로 (DATA & TRACKING PATHS)
*데이터베이스 파일 위치입니다. (변경하지 않는 것을 권장합니다).*
* **`DownloadListPath`**: 대기열 텍스트 파일 경로 (`".\Data\download_list.txt"`).
* **`CatalogCsvPath`**: 카탈로그 데이터베이스 CSV 경로 (`".\Data\series_catalog.csv"`).

## 📝 6. 로깅 시스템 (LOGGING SYSTEM)
*스위트가 기록하는 로그 파일의 수준을 조절합니다.*
* **`LogLevel` / `CrawlerLogLevel`**
  * **기본값:** `"Verbose"` (모든 작업 기록). 용량을 아끼려면 `"Info"`, `"Warn"`, `"Error"`로 변경하세요.
* **`CrawlerLogMaxMB` / `CrawlerLogMaxFiles`**
  * **기본값:** `10` MB / `5` 개. 로그 파일이 무한정 커지지 않도록 자동으로 분할하고 오래된 로그를 삭제합니다.

## ⏰ 7. 스케줄러 설정 (SCHEDULER SETTINGS)
*백그라운드 자동화의 핵심 설정입니다.*
* **`AutoCrawlerIntervalHours`**
  * **기본값:** `1`
  * **기능:** 백그라운드 스케줄러가 최신 글을 확인하기 위해 깨어나는 주기(시간 단위)입니다.
* **`BoardCrawlerIntervalHours`**
  * **기본값:** `12`
  * **기능:** 다소 무거운 작업인 '게시판 시리즈 크롤링'이 백그라운드에서 실행되는 주기입니다.

## 🛠️ 8. 시스템 고급 설정 (ADVANCED / SYSTEM)
* **`ForceLegacyMode`**
  * **기본값:** `False`
  * **기능:** 시스템에 최신 PowerShell(pwsh 7+)이 설치되어 있더라도, 강제로 기본 Windows PowerShell(v5.1)을 사용해 런처를 구동하도록 만듭니다. 호환성 문제가 있을 때만 `True`로 켜세요.
